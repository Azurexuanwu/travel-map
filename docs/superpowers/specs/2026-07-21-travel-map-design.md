# 中国旅游足迹地图 — 设计文档

## 概述

一个可在线访问的中国地图网页，标记去过的城市并展示旅游照片。支持密码登录后管理（增删城市和照片），未登录时只读查看。地图使用 ECharts + 中国 GeoJSON，后端数据存储在 Supabase 免费层，前端部署在 GitHub Pages。

## 功能清单

### 只读（访客）

1. 查看中国地图，已标记城市以散点/标记展示
2. 点击城市，地图上弹出气泡，显示照片轮播（左右翻看 + 照片描述）
3. 响应式布局，桌面和移动端均可用

### 编辑（密码登录后）

4. 管理面板（侧边抽屉），列出所有已标记城市
5. 添加新城市：选择城市名（预设列表 + 经纬度）或自定义输入经纬度
6. 上传照片：每个城市可上传多张，每张可附描述文字
7. 删除单张照片
8. 删除整个城市及其所有照片

## 技术架构

```
travel-map.html (GitHub Pages)
├── ECharts 5 (CDN) — 地图渲染
├── echarts/map/js/china.js (CDN) — 中国 GeoJSON
├── @supabase/supabase-js (CDN) — 数据读写
└── Supabase 后端
    ├── cities 表 — 城市数据（id, name, lat, lng）
    ├── photos 表 — 照片元数据（id, city_id, url, description）
    └── Storage bucket — 照片文件
```

### 数据表设计

**cities 表**
| 列 | 类型 | 说明 |
|---|---|---|
| id | uuid (PK) | 主键 |
| name | text | 城市名称 |
| lat | float8 | 纬度 |
| lng | float8 | 经度 |
| created_at | timestamptz | 创建时间 |

**photos 表**
| 列 | 类型 | 说明 |
|---|---|---|
| id | uuid (PK) | 主键 |
| city_id | uuid (FK → cities) | 所属城市 |
| storage_path | text | Supabase Storage 中的文件路径 |
| description | text | 照片描述 |
| sort_order | int2 | 排序序号 |
| created_at | timestamptz | 创建时间 |

### 认证

简单密码验证：
- 在 Supabase 中存储一个简单的凭据配置
- 前端在 localStorage 中保存登录态（密码 hash）
- 进入管理模式需输入密码
- 读取操作使用 Supabase 匿名 key（RLS 策略：读公开，写需认证）

## 交互流程

### 查看照片
```
点击地图上的城市标记 → ECharts 点击事件 → 获取城市坐标
→ 计算气泡浮层位置（地图 canvas 坐标 → 页面坐标）
→ 渲染气泡 HTML（第一张照片 + 描述 + 翻页箭头）
→ 点击左右箭头 → 切换当前照片索引 → 更新气泡内容
→ 点击地图空白处 / 关闭按钮 → 隐藏气泡
```

### 管理城市
```
点击「管理」→ 输入密码 → 验证通过 → 打开侧边抽屉
→ 显示城市列表（从 Supabase 加载）
→ 「添加城市」: 弹出表单 → 选城市/输入经纬度 → 上传照片（多选+描述）
→ 上传过程: FileReader 读文件 → 上传到 Supabase Storage → 写入 photos 表
→ 「删除城市」: 确认 → 删除 photos 记录 + storage 文件 → 删除 city 记录
→ 操作完成后刷新地图上的标记
```

### 密码管理
```
首次访问 → localStorage 无登录态 → 只读模式
点击「管理」→ 弹出密码输入框 → 与 Supabase 存储的秘密比对
→ 正确: 存 hash 到 localStorage，切换为编辑模式
→ 错误: 提示密码错误
```

## 部署步骤

1. 注册 Supabase 账号，创建项目
2. 在 Supabase SQL Editor 中执行建表 SQL + RLS 策略
3. 创建 Storage bucket `travel-photos`
4. 将 Supabase URL 和 anon key 填入 `travel-map.html` 配置区
5. 推送到 GitHub 仓库，启用 GitHub Pages
6. 访问 `https://<username>.github.io/travel-map/`

## 预设城市列表

代码中内置一份常见旅游城市 + 经纬度，方便快速添加：北京、上海、广州、深圳、成都、重庆、杭州、南京、西安、武汉、长沙、厦门、青岛、大连、哈尔滨、昆明、丽江、大理、桂林、三亚、拉萨、乌鲁木齐、呼和浩特、西宁、兰州、银川、海口、贵阳、南昌、合肥、郑州、济南、太原、石家庄、天津、苏州、无锡、宁波、福州、珠海。

## 边界与约束

- 照片单张 ≤ 2MB（前端校验 + Supabase Storage 限制）
- 每个城市最多 20 张照片（前端限制，避免气泡内翻页过长）
- 气泡在移动端自适应缩小，确保不超出屏幕
- 密码使用 SHA-256 hash 比对，不存明文（Supabase 中存 hash 值）
- 不上传照片时，气泡显示占位图标
